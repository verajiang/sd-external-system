## 1. 执行摘要

智播星商家端是面向商家团队的 AI 视频创作 SaaS。产品帮助商家基于商品图、参考视频、音频和已授权真人形象快速生成电商短视频，并提供团队成员、席位、额度和素材授权管理能力。当前版本聚焦三个一级模块：生成、资产、团队；不提供首页、模板中心、独立视频作品页、用量一级菜单或设置页。管理员负责团队、额度、资产授权与全量记录管理；普通账号负责使用被分配额度完成自己的视频生成和素材使用。

## 3. 问题陈述

### 3.1 谁有这个问题

主要用户是需要批量制作商品短视频的商家团队，包括运营负责人、内容负责人、投放/直播团队成员和素材管理员。

### 3.2 问题是什么

商家团队在制作短视频时，需要同时处理商品素材、参考视频、真人授权、团队成员、额度消耗和结果复用。如果系统只提供单点生成能力，团队很难判断哪些素材可用、哪些真人形象已授权、哪个成员消耗了多少额度，以及如何复用成功生成结果。

### 3.3 为什么痛

- 素材状态不清会导致不可用、未授权或审核失败素材被误用。
- 真人形象涉及肖像授权和素材审核，需要从创建、认证、上传、审核到引用形成闭环。
- 团队协作下，管理员需要控制席位、成员权限、月度额度和批量生成能力。
- 普通账号不应看到团队和授权管理数据，只需要清楚自己能用多少额度、能用哪些素材、自己的生成结果。

## 4. 目标用户与角色

### 4.1 管理员

管理员代表商家组织的运营负责人或内容负责人。

核心目标：

- 管理团队成员、席位和成员额度。
- 查看所有账号生成记录和子账号消耗。
- 管理普通素材、真人形象、授权记录。
- 控制哪些真人素材可以用于生成。

### 4.2 普通账号

普通账号代表商家团队中的内容执行人员。

核心目标：

- 使用被分配额度生成视频。
- 查看自己的生成记录和作品。
- 使用可用共享素材和自己上传的素材。
- 不接触团队管理、授权记录、席位和套餐升级入口。

## 5. 产品目标

1. 让商家可以在一个工作台内完成素材选择、Prompt 输入、参数配置、视频生成和结果复用。
2. 让管理员可以统一管理素材、真人形象、授权记录、成员、席位、消耗和额度。
3. 让普通账号只看到与自己执行相关的信息，降低误操作和越权风险。
4. 输出可交付给前端技术的高保真 HTML 原型，作为页面开发参考。

## 8. 核心流程

### 8.1 视频生成流程

1. 用户进入“生成”页。
2. 用户选择生成模式，当前仅支持：
   - 全能参考
   - 首尾帧
3. 用户上传本地素材，或从资产素材库选择可用素材。
4. 用户在输入框中输入 Prompt。
5. 用户输入 `@` 时，只展示当前已上传或已选择的素材。
6. 用户配置参数：
   - 生成模式
   - 比例
   - 清晰度
   - 时长
   - 生成条数
   - 输出声音
7. 用户点击生成。
8. 系统新增生成任务，状态先为“生成中”。
9. Mock 几秒后任务变为“生成成功”。
10. 用户可重新编辑、再次生成、下载、收藏。

验收标准：

- 默认进入页是“生成”。
- 默认生成模式是“全能参考”。
- 生成模式只能是“全能参考 / 首尾帧”。
- `@` 只展示当前输入上下文中的素材，不展示全量资产库。
- 资产库弹窗只展示可用素材。
- 一次生成多条结果时，对话流中展示一张任务卡，作品视图拆成多个作品卡。
- 普通账号只看到自己创建的生成记录。
- 普通账号生成时使用本人剩余额度判断是否可提交。

### 8.2 资产管理流程

资产页 Tab：

- 全部资产
- 真人形象
- 图片
- 视频
- 音频
- 授权记录，仅管理员可见

普通素材能力：

- 上传图片、视频、音频。
- 预览素材。
- 改名素材。
- 删除素材。
- 不提供普通素材详情页。
- 不提供普通素材“引用”按钮。

真人形象能力：

- 管理员创建真人形象。
- 步骤为：填写信息 / 发送认证 / 上传素材 / 等待审核。
- 填写信息只保留形象名称、授权时间。
- 一个真人形象可以有多个同人素材。
- 只有已认证且素材状态为可用的真人素材可用于生成页引用。
- 真人形象详情只展示素材，不展示基础信息大面板。
- 失败素材展示失败原因，并支持管理员删除。

验收标准：

- 普通资产上传不包含真人素材。
- 真人素材只能从真人形象流程上传。
- 处理中、失败、未认证素材不能出现在生成页资产选择弹窗中。
- 普通账号不可见“授权记录”Tab。
- 普通账号可见可用共享素材和自己上传素材。
- 普通账号对非本人共享素材只能预览，不能改名/删除。

### 8.3 团队管理流程

团队页仅管理员可见。

团队页 Tab：

- 成员管理
- 席位管理
- 子账号消耗
- 成员额度

成员管理：

- 管理员添加成员。
- 添加成员时填写手机号、选择账号类型。
- 添加成功后系统生成初始密码。
- 管理员复制初始密码给成员。
- 角色仅支持管理员、普通账号。
- 成员操作包括设为管理员、停用/启用、重置密码。

席位管理：

- 展示当前套餐、到期时间、总席位、已用席位、剩余席位、剩余额度、席位使用进度。
- 只保留“加购席位”。
- 不提供“查看套餐”。

子账号消耗：

- 展示本月总生成、总消耗额度、成功生成、失败生成、已下载作品、人均消耗。
- 展示全部子账号消耗排行，不限制 Top 5。
- 成员排行列表必须展示表头，字段包括排名、成员、本月生成、消耗额度、成功生成、失败生成、下载/额度。
- 消耗明细作为独立对账板块展示，必须展示表头，字段包括时间、账号、模式、视频名称、真人形象、消耗额度、状态、下载状态。
- 消耗明细支持导出明细和生成对账结果，用于财务或运营核对额度消耗。

成员额度：

- 支持“不限制 / 按成员限制”。
- 支持设置成员月度额度。
- 支持设置是否允许批量生成。
- 支持设置单次最多生成条数。

验收标准：

- 普通账号不显示团队菜单。
- 普通账号访问团队 hash 时回到生成页。
- 套餐弹层中的“查看用量明细”跳转到团队页“子账号消耗”Tab。
- 普通账号不显示升级套餐按钮，只显示自己的剩余额度、本月分配、本月已用。

## 10. 数据模型需求

核心对象：

- Merchant：企业套餐、剩余额度、已用额度、席位、到期时间。
- User：账号名、角色。
- Asset：素材名称、类型、状态、上传人、失败原因、使用次数、创建时间。
- HumanProfile：真人形象状态、素材数量、失败原因。
- AuthorizationRecord：授权对象、关联真人、状态、期限、文件名。
- TeamMember：成员、角色、状态、月度生成、消耗额度、下载数、额度限制、批量权限、单次生成上限。
- GenerationItem：Prompt、引用素材、参数、状态、结果、创建人、消耗额度、失败原因。

后端落地时需要将当前 mock data 替换为真实 API，并保持前端状态枚举一致。

## 11. 成功指标

产品验证指标：

- 首次生成完成率：新商家进入系统后完成至少一次生成的比例。
- 素材可用率：上传素材中最终可用于生成的比例。
- 真人认证通过率：创建真人形象后成功认证并有可用素材的比例。
- 生成成功率：生成成功任务数 / 总生成任务数。
- 结果复用率：重新编辑、再次生成、下载行为占生成成功任务的比例。
- 团队额度命中率：成员额度设置后，普通账号超额生成被拦截的比例。

原型验收指标：

- 管理员版和普通账号版均可通过单 HTML 打开。
- `?account=member` 可进入普通账号视角。
- 无外部 JS/CSS/logo 依赖。
- 生成、资产、团队核心交互均可在 mock 环境演示。

## 14. 风险与依赖

### 14.1 真实后端接口风险

当前原型使用 mock data。真实落地时，需要后端提供生成任务、资产、真人认证、授权记录、团队成员、额度消耗等接口，否则前端无法按当前流程完整闭环。

缓解：

- 在接口设计阶段保持当前类型枚举和页面状态一致。
- 先实现查询和状态流转接口，再接真实上传和生成服务。

### 14.2 真人授权合规风险

真人形象涉及肖像授权、实名认证和素材审核。若授权链路不完整，可能导致不可用素材被误引用。

缓解：

- 后端必须区分真人形象认证状态和素材审核状态。
- 生成页只允许选择状态为可用的素材。
- 授权过期或认证失败时，应自动从可用范围移除。

### 14.3 普通账号越权风险

普通账号不应查看团队成员、授权记录、套餐、全量生成记录等管理员数据。

缓解：

- 前端隐藏只是体验层，后端接口必须按账号角色做权限校验。
- 普通账号生成记录查询必须按 creator/user_id 过滤。
- 资产操作必须校验上传人或管理权限。

## 15. 开放问题

1. 普通账号是否可以上传普通素材后自动共享给全团队，还是默认仅本人可见？
2. 真人形象素材是否允许普通账号提交，由管理员审核后进入真人形象？
3. 成员额度的周期是自然月、套餐月，还是企业自定义账期？
4. 生成失败是否退还额度？
5. 下载是否消耗额度，还是仅生成消耗额度？
6. 套餐升级入口是否需要接入商务联系、在线支付或线索表单？
7. 授权记录是否需要到期提醒和自动禁用关联真人素材？
8. 首尾帧模式是否必须强制上传两个图片素材位？

## 16. 里程碑建议

### M1：前端静态原型确认

- 确认信息架构、角色权限、核心页面和设计系统。
- 交付单 HTML 原型。

### M2：接口定义与权限模型

- 定义用户、团队、资产、真人形象、授权、生成任务、额度消耗接口。
- 明确管理员和普通账号权限。

### M3：核心生成闭环

- 接入真实素材上传。
- 接入真实生成任务提交、状态轮询、结果展示、下载。

### M4：资产与真人授权闭环

- 接入普通素材管理。
- 接入真人认证链接、素材审核、授权记录。

### M5：团队与额度闭环

- 接入成员、席位、成员额度、子账号消耗。
- 完成普通账号额度限制和越权拦截。
